Prerequisites

Before installing engram, make sure you have the following:

  • * Node.js 20 or higher — check with node --version. Node 22 LTS recommended.
  • * npm — ships with Node.js. Check with npm --version.
  • * Claude Code — the CLI agent from Anthropic. engram's Sentinel hooks integrate directly with Claude Code's hook system.
  • ~ Git (optional) — enables the git miner (commit history analysis) and auto-rebuild hooks. Not required for the core graph.
  • ~ Cursor 1.7+ (optional) — experimental Cursor adapter available via engram cursor-intercept.
Zero dependencies

engram has zero native dependencies. No Python, no Rust, no build tools, no Docker, no cloud services. Everything runs locally in a single SQLite file.

Tier 1 — Build the Graph

STEP 2

Initialize your project

Navigate to your project directory and run:

cd ~/my-project
engram init

This scans your codebase using tree-sitter AST extraction (no LLM, zero tokens) and builds a structural knowledge graph at .engram/graph.db.

📊 Token savings: 6.4x fewer tokens vs relevant files (25x vs full corpus)

What you can do now

engram query "how does authentication work"   # query the graph
engram gods                                       # show most-connected entities
engram stats                                      # node/edge counts
engram path "auth" "database"                     # shortest path between concepts
engram gen                                        # generate CLAUDE.md section

Tier 2 — Install Sentinel Hooks

This is where engram becomes invisible infrastructure. The Sentinel layer intercepts Claude Code tool calls at the hook boundary — before the agent even sees the file.

STEP 3

Wire Sentinel into Claude Code

engram install-hook

This writes hook entries into .claude/settings.local.json (gitignored by default). You can preview first:

engram install-hook --dry-run    # see what would change

What Sentinel does automatically

Agent calls Read
→
Sentinel intercepts
→
Graph has coverage?
→
~300 token summary
→
Agent uses summary

Seven hook handlers compose the interception stack:

  • PreToolUse:Read — blocks file reads and delivers structural summaries when graph has ≥70% confidence
  • PreToolUse:Edit / Write — injects landmine warnings before edits that touch files with known mistakes
  • PreToolUse:Bash — intercepts cat, head, tail commands that bypass Read
  • SessionStart — injects a project brief at the start of every session
  • UserPromptSubmit — pre-queries the graph based on your message keywords (TF-IDF filtered)
  • PostToolUse:Read — observes fallback reads for analytics

Scope options

engram install-hook                    # local scope (default, gitignored)
engram install-hook --scope project    # committed to repo, shared with team
engram install-hook --scope user       # global ~/.claude/settings.json
Important

The --scope user option installs hooks globally. This means every Claude Code session will run through engram — even projects that haven't been initialized. engram handles this gracefully (passes through if no .engram/ exists), but local scope is recommended for most users.

Tier 3 — Index Your Skills

STEP 4

Include skills in the graph

engram init --with-skills

This re-scans the project and also indexes your ~/.claude/skills/ directory. Skills appear as concept nodes in the graph, linked by triggered_by edges to their keyword triggers.

Queries now surface relevant skills alongside code structure. Example: querying "authentication" might return both your src/auth.ts module and the better-auth skill.

Custom skills path

Pass a custom path: engram init --with-skills /path/to/skills

Tier 4 — Auto-Rebuild on Commit

STEP 5

Install git hooks

engram hooks install

This installs a post-commit git hook that automatically rebuilds the graph after every commit. Your graph never goes stale — and it costs zero tokens.

The full setup (copy & paste)

# One-time setup per project (all 4 tiers)
npm install -g engramx
cd ~/my-project
engram init --with-skills      # Tier 1 + 3: graph + skills
engram install-hook            # Tier 2: Sentinel hooks
engram hooks install           # Tier 4: auto-rebuild

That's it. Open a Claude Code session and start working. engram runs invisibly.

Optional — MEMORY.md integration

engram gen --memory-md

Writes a marker-bounded block of structural facts into Claude's native MEMORY.md. Claude's Auto-Dream owns the prose; engram owns the structure. They complement each other — engram never touches content outside its markers.

The graph is built by three miners that run locally with zero LLM cost:

  • AST miner — tree-sitter extraction across 10 languages. Functions, classes, imports, exports, call relationships.
  • Git miner — commit history analysis. Hot files, co-change patterns, authorship.
  • Session miner — learns from your Claude Code conversations. Mistakes, decisions, patterns.

Troubleshooting

engram: command not found

The global npm bin directory isn't in your PATH. Fix:

# Find where npm puts global bins
npm config get prefix
# Add to your shell profile (~/.zshrc or ~/.bashrc)
export PATH="$(npm config get prefix)/bin:$PATH"
engram init finds 0 nodes

engram uses tree-sitter grammars bundled with the package. Supported languages: TypeScript, JavaScript, Python, Go, Rust, Java, C, C++, C#, Ruby. If your project uses an unsupported language, the AST miner will find file-level nodes but no function/class-level detail.

Also check: are you running engram init from the project root (where package.json or .git/ lives)?

Sentinel isn't intercepting reads

Three common causes:

  1. Hooks not installed — run engram install-hook and check .claude/settings.local.json for engram entries.
  2. Confidence below threshold — engram only intercepts when graph coverage is ≥70%. New or poorly-covered files pass through. Run engram hook-preview <file> to check.
  3. Kill switch enabled — check for .engram/hook-disabled. Remove it or run engram hook-enable.
Graph is stale after code changes

Run engram init again to rebuild the graph. Or install Tier 4 git hooks (engram hooks install) to auto-rebuild on every commit.

The Sentinel layer also checks file modification times — if a file changed since the last mine, the hook passes through instead of serving a stale summary.

Windows: paths show as src\auth.ts instead of src/auth.ts

Fixed in v0.4.1. The graph now stores all paths in POSIX form regardless of host OS. If you initialized on an older version, run engram init again to rebuild with normalized paths.

How do I use engram with the MCP server?

engram ships an MCP server for direct integration with Claude Code, Windsurf, or any MCP client:

# In .claude.json (per-project MCP config)
{
  "mcpServers": {
    "engram": {
      "command": "engram-serve",
      "args": []
    }
  }
}

This exposes 6 tools: query_graph, shortest_path, god_nodes, graph_stats, learn_context, list_mistakes.

The MCP server is complementary to Sentinel — Sentinel intercepts automatically, while the MCP server lets the agent query explicitly when it wants deeper structural context.

Uninstall

engram's uninstall is surgical. Each layer can be removed independently.

engram uninstall-hook            # remove Sentinel hooks (preserves other hooks)
engram hooks uninstall           # remove git hooks
rm -rf .engram/                  # delete graph database
npm uninstall -g engramx         # remove the CLI
